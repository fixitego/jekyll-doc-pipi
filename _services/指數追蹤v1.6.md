# 表格美化與可讀性規則

為避免 Markdown 表格因單格文字過長而擠在一起，所有表格需遵守以下規則：

1. 日期、狀態、數值、漲跌、說明分欄呈現，不要把日期與數值全部塞在同一格。
2. 「客觀原因」與「客觀事件」每格最多 1～2 句。
3. 若原因太長，請縮短為事實重點。
4. 不要在同一格放入過多標的，美國 AI / 半導體代表股表現請拆成多列。
5. 表格欄位名稱要短，避免欄位過寬。
6. 數值欄只放數值，不放長句。
7. 若需要標示資料狀態，使用獨立欄位，例如：`狀態`、`資料日`、`盤中價`。
8. 避免使用「最新」作為混合欄位，因為最新可能是收盤，也可能是盤中。

# 資料時間與比較基準規則

由於本晨報固定於 **台灣時間 08:30（UTC+8）** 產出，請依各市場交易時段採用適當資料，並清楚標示資料時間。

## 核心比較原則

不論市場是否已開盤，**主要漲跌幅欄位一律使用「最近一次完整收盤」與「再前一次完整收盤」比較**。

也就是：

| 情境     | 主要比較方式                          |
| ------ | ------------------------------- |
| 美股已收盤  | 最新收盤 vs 前一交易日收盤                 |
| 台股尚未開盤 | 前一交易日收盤 vs 再前一交易日收盤             |
| 日本已開盤  | 前一交易日收盤 vs 再前一交易日收盤             |
| 韓國已開盤  | 前一交易日收盤 vs 再前一交易日收盤             |
| 商品盤中交易 | 最近一次結算價 / 收盤價 vs 前一交易日結算價 / 收盤價 |

---

## 盤中資料處理原則

若市場在 08:30 已經開盤，例如日本、韓國，盤中資料不得取代主要收盤比較。

盤中資料請放在額外欄位：

* `盤中時間`
* `盤中價`
* `盤中較前收`

其中：

* `收盤`、`前收`、`漲跌`、`漲跌幅`：反映最近一次完整交易日的收盤變化。
* `盤中價`、`盤中較前收`：反映今日開盤後盤中狀態。

---

## 日期標示規則

建議欄位：
